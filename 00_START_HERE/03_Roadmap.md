## Content
Phase 0: Product definition.

- Lock product docs.
- Define non-negotiables.
- Define screen map.
- Draft schema and quest logic.

Phase 1: Foundation.

- Create app structure.
- Configure Telegram Mini App basics.
- Configure Supabase.
- Define initial schema.
- Set theme tokens and navigation shell.

Phase 2: Core loop.

- Onboarding.
- Profile and goals.
- Path selection.
- Daily quest generation and completion.

Phase 3: Logging engine.

- Workouts, meals, water, sleep, learning, language, weekly check-ins.

Phase 4: Progression engine.

- XP, levels, ranks, streaks, achievements, domain scoring.

Phase 5: Telegram integration.

- Bot notifications, reminders, streak alerts, weekly prompts, level/rank alerts.

Phase 6: Polish.

- Motion, visual quality, mobile optimization, data consistency, edge cases.
